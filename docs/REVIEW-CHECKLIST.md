# WASTE Journal 审核清单

**审核员专用** - 确保每篇投稿都符合 WASTE 标准

---

## 📋 初审清单（5 分钟内完成）

### 格式检查
- [ ] 使用投稿模板
- [ ] 元数据完整（标题、作者、分类、标签）
- [ ] Markdown 格式正确
- [ ] 无明显排版问题

### 基本要求
- [ ] 字数在 300-5000 字范围内
- [ ] 语言通顺（中文/英文）
- [ ] 有明确主题
- [ ] 非纯广告/推广内容

### 分类判断
| 分类 | 判断标准 |
|------|----------|
| 🐛 bug-report | 神秘报错、环境问题、"Works on My Machine" |
| 🏺 archaeology | 2015 年前技术、经典代码、历史考古 |
| 🎨 art | 硬件翻车、设计灾难、失败创意 |
| ⚫ black-hole | 无用脚本、资源浪费、跑分程序 |

**初审结果**：
- ✅ 通过 → 进入复审
- ⚠️ 需修改 → 给出修改建议
- ❌ 拒绝 → 礼貌回复 + 原因

---

## 🔍 复审清单（15-30 分钟）

### 内容质量
- [ ] 有技术价值或娱乐价值
- [ ] 论述清晰、逻辑连贯
- [ ] 代码示例可运行（如适用）
- [ ] 图片/图表清晰（如适用）

### WASTE 风格
- [ ] 符合"学术幽默"调性
- [ ] 表面正经、内含荒诞
- [ ] 不过于严肃/说教
- [ ] 有一定"无用"特质

### 原创性检查
- [ ] 非抄袭内容
- [ ] 引用已标注来源
- [ ] 无版权问题
- [ ] 未在他处发表（或已获授权）

### 技术准确性
- [ ] 技术描述准确
- [ ] 代码无明显错误
- [ ] 数据/引用可靠
- [ ] 无误导性信息

**复审结果**：
- ✅ 接受 → 进入编辑流程
- ⚠️ 修改后接受 → 列出修改要求
- ❌ 拒绝 → 详细说明原因

---

## ✍️ 编辑清单（10-20 分钟）

### 标题优化
- [ ] 吸引眼球
- [ ] 准确反映内容
- [ ] 长度适中（10-30 字）
- [ ] 无标题党嫌疑

### 格式统一
- [ ] 标题层级清晰（H1/H2/H3）
- [ ] 代码块有语言标识
- [ ] 图片有描述文字
- [ ] 链接可访问

### 内容优化
- [ ] 摘要精炼（100-200 字）
- [ ] 段落长度适中
- [ ] 重点内容突出
- [ ] 无错别字/语法错误

### SEO 优化
- [ ] 标签完整（3-5 个）
- [ ] 分类准确
- [ ] 描述包含关键词
- [ ] 便于检索

---

## 🚀 发布前检查

### 最终确认
- [ ] 作者确认最终稿
- [ ] 所有检查项通过
- [ ] 无遗留问题
- [ ] 发布时机合适

### 技术检查
- [ ] 构建无错误
- [ ] 页面显示正常
- [ ] 链接无 404
- [ ] 图片加载正常

### 发布后跟进
- [ ] 通知作者
- [ ] 社交媒体同步
- [ ] 收集读者反馈
- [ ] 统计数据记录

## 🎯 特殊情况处理

### 争议投稿
- 两位审核员意见不一致
- 处理：第三位审核员仲裁

### 敏感内容
- 涉及隐私/安全/法律
- 处理：谨慎评估，必要时拒绝

### 紧急投稿
- 时效性强的内容
- 处理：优先审核，24 小时内发布
